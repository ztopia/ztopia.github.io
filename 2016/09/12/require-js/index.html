导入require.js

->定义模块 define([],function(){}) //定义模块

-> 调用模块 require([‘tab’],function(){}) –>路径应该是一个字符串格式（tab.js）

-->路径不需要后缀名(requirejs只能加载js文件)
-->这样就能执行了tab.js模块里面的代码

是否支持AMD规范

搜索define.amd

不支持AMD规范肯定没有返回值

如果支持AMD规范，分析该功能的入口，看看该入口是不是由该文件自己提供的

jquery插件支持AMD规范但是没有返回值 ，给返回值的目的是给一个功能入口（没有返回值，不能使用依赖注入）

if(typeof define==’function’ && define.amd){

​ define([],function(){

​ return v8

​ })

​ window.v8=v8;

}

在第三方模块中加入以上代码就支持了amd规范

文件路径

1.直接调用文件（优先级最低）

注意调用文件时的路径

2.通过入口文件调用（优先级第二高）

此时文件路径基于js文件夹

3.通过自定义文件定义路径（优先级最高）

require.config({

    //配置基础路径        -->以后所有的模块路径都是基于该基础路径的
    baseUrl:"js"      //相对于index.html的路径
    paths:{
      'jquery':'lib/jquery-1.2.4'   //配置特定文件的路径，基于baseUrl
      'asset':'asset'           //配置指定文件夹路径

    }
    shim:{

    }
});
